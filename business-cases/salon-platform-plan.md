## üîê Google Calendar Twee-Weg Sync Architectuur

### Authenticatie Flow
```
1. Medewerker klikt "Koppel Google Calendar"
2. OAuth2 redirect naar Google
3. Gebruiker geeft toestemming voor calendar.events scope
4. Callback met auth code
5. Exchange voor access + refresh tokens
6. Opslaan in GoogleCalendarSync tabel
```

### Salonized ‚Üí Google (Outbound)
```
Nieuwe afspraak in SalonPro:
1. Appointment create/update trigger
2. Check of employee.googleSyncEnabled = true
3. Maak Google Calendar event via API
4. Sla googleEventId op in Appointment
```

### Google ‚Üí SalonPro (Inbound) ‚≠ê DIT MIST BIJ SALONIZED
```
Nieuwe/gewijzigde event in Google Calendar:
1. Google Push Notification (webhook)
2. Ontvang op /api/webhooks/google-calendar
3. Fetch event details via API
4. Match met bestaande appointment OF maak "geblokkeerd" slot
5. Real-time update in UI via WebSocket/polling
```

### Conflict Handling
```
Als Google event overlapt met bestaande afspraak:
1. Toon waarschuwing in dashboard
2. Email notificatie naar salon eigenaar
3. Optie: automatisch weigeren √≥f markeren voor review
```
